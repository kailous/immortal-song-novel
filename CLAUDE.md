# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 每次创作会话的第一件事

**按顺序读这四个文件，然后再做任何事：**

1. `_工作台/当前状态.md` — 剧情进度、角色状态、上次断点
2. `_工作台/铁律速查.md` — 创作红线速查（禁用词、世界观铁律、对白语调）
3. `_工作台/角色速查.md` — 三位写手核心指令 + 智囊团 + 范文库索引（**替代逐一读取独立档案**）
4. `_工作台/章节简报_模板.md` — 动笔前填写，锁定本章任务

> 这四步是创作会话的"冷启动协议"，跳过它们会导致设定冲突。
> 写手/智囊团的独立档案（`写手/*.md`、`智囊团/*.md`）只在需要深挖细节时才读取。

## 常用命令

```bash
make publish       # 将正文 Markdown 发布到网站 JSON（章节定稿后必须执行）
make sync-chars    # 同步角色轻索引到网站，并公开 Markdown 档案
make check         # 扫描正文与设定库的逻辑冲突
make report        # 生成设定库简要报告
make all           # publish + sync-chars
```

## 目录职责

| 目录 | 职责 |
|------|------|

## 设定库核心规范

**`设定库/` 与 `大纲/` 的分界线**：
- 设定库 = 已在正文中发生的事实（Log）
- 大纲 = 尚未发生的规划、未来路线图

**角色档案必须包含七模块**：`〇姓名渊源 → 一形象 → 二背景 → 三性格 → 四作用 → 五装备 → 附生平志`

**设定库分类（严禁平铺存放）**：
- `01_底层逻辑与物理机制`：量子重置、手环机制、物理常数
- `02_社会演进与制度体系`：政治架构、法律铁律、监察体系
- `03_群系生态与情感锚点`：后宫生态、阶层关系、基础导览

## 正文 → 网站发布流程

```
正文/第N章_标题.md 定稿
  → 更新 _工作台/当前状态.md
  → 更新 设定库/00_全局时间线.md
  → 更新 相关角色生平志（附节末尾）
  → make publish   → 生成 docs/chapters/chapter-N.json + index.json
  → make sync-chars（如有角色变动）
```

---

## 网站架构（`docs/`）

纯静态站点，无构建步骤，直接部署 GitHub Pages。

- 章节内容：`docs/chapters/chapter-N.json`，由 `reader.js` 通过 `fetch()` 加载
- 角色索引：`docs/data/characters_zh.json` / `docs/data/characters_en.json`，列表页按语言读取；中文详情页直读 `docs/content/profiles/zh/*.md`，英文详情页直读 `docs/content/profiles/en/*.md`
- 设计主题：深色 `#0a0b0f` + 古金色 `#c8a86e`；正文 `Noto Serif SC`，界面 `Inter`
