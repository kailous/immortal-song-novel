# Setting Art Generator Skill

此 Skill 专门服务于《长生不死》的**设定图**工作流，目标是稳定产出可复用的角色、种族、装备、建筑与文明档案图。

## 当前公开设定图基线
- 默认画幅：**1:1 正方形**
- 默认用途：图鉴 / 档案图，而不是海报
- 默认构图：主体居中，正面或近正面，结构清楚，可直接用于详情页
- 默认背景：低干扰深色背景或极简环境背景
- 默认风格：写实、克制、可读性优先
- 默认输出：**纯图像，不带文字标注、不带面板说明、不带 UI 边框**
- 现有公开设定图规律见 `references/style_baseline.md`

## 工作边界
- 服务对象是**设定图**，不是章节剧情插图。
- 默认产出单张静态图，优先考虑正面展示、结构清楚、设定可读。
- 若用户未指定画幅，默认输出 **1:1**。
- 支持两条路径：
  - **正向生成**：从设定文档生成提示词与设定图
  - **反推重绘**：从现有图片提炼视觉特征，再结合世界观修正重绘
- 设定图优先保证**信息准确性**，其次才是风格化。
- 若存在现有官方公开图，重绘时必须说明：
  - 哪些视觉元素保留
  - 哪些元素修正
  - 为什么修正

## 适用场景
- 角色立像
- 外星种族设定图
- 道具档案图
- 装备/制服设定图
- 建筑或文明风貌图
- 根据现有图片反推 prompt 后重绘

## 标准流程

### 第一步：锁定设定目标
先明确：
1. 画面对象是什么
2. 用途是图鉴、档案页还是宣传图
3. 是正向生成还是反推重绘
4. 必须保留的世界观事实是什么
5. 允许修正的视觉问题是什么
6. 是否维持公开图鉴基线（默认是）

### 第二步：整理结构化 brief
优先使用 `references/brief_template.md`，至少写清：
- 对象名称
- 图像类型
- 画幅（默认 `1:1`）
- 视角/构图
- 生理结构
- 材质
- 服装/装备
- 环境
- 光线
- 必须保留元素
- 必须避免元素
- 参考来源

### 第三步：反推分析（如果有参考图）
对现有图先拆成三层：
- **保留项**：脸型、体型、轮廓、主配色、核心器官/义体位置
- **问题项**：与设定冲突、结构错误、风格跑偏
- **强化项**：图里没讲清，但设定里必须补上的信息

并且要先对照 `references/style_baseline.md`，判断旧图是否符合当前设定图系统。

不要直接照抄参考图表面描述，要回到设定文本核对。

### 第四步：生成提示词
优先用 `scripts/build_setting_prompt.py` 从 JSON brief 生成：
- 主提示词
- 负面约束
- 设定核对点

### 第五步：审核提示词
按 `references/review_checklist.md` 检查：
- 是否符合设定事实
- 是否保留了正确的参考特征
- 是否修掉了旧图错误
- 是否把设定图画成剧情海报
- 是否偏离了现有公开图鉴的正方形档案图体系
- 是否错误生成了文字、说明面板、信息图边框
- 是否出现人类化、偶像化、游戏概念图化倾向

### 第六步：生成与复核
生成后重点看：
- 结构是否正确
- 义体/装备位置是否正确
- 种族辨识度是否稳定
- 画面是不是“像设定图”，而不是“像电影剧照”

## 设定图原则

### 1. 信息优先
- 设定图先讲清对象，再追求氛围。
- 正面、半身、全身、三视图式表达都比花哨构图更有价值。

### 2. 结构优先
- 外星种族先确保骨架、肢体、义体逻辑准确。
- 道具先确保材料、接口、尺寸逻辑准确。

### 3. 风格约束
- 默认风格：**写实设定图 / 高级科幻档案图**
- 默认画幅：**1:1**
- 不是动漫立绘
- 不是游戏角色卡
- 不是章节海报

- 参考一致性：通过 / 不通过

## 注意事项
- 若用户要求“重绘现有设定图”，必须先说明旧图哪里不准。
- 若对象已有公开图，默认把它当**官方基线**，不是完全自由重做。
- 若是塔伦人这类高设定密度对象，先读对应 profile 再写 prompt。
