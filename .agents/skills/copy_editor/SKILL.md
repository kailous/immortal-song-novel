# Copy Editor Skill

此 Skill 用于中文内容审校，适用于正文、设定、图鉴、词典和其他文稿类文件。

## 工作边界
- 先审查，再修改。
- 第一轮只输出问题清单，不直接改文件。
- 问题清单必须编号，便于用户按编号确认。
- 每条问题至少包含：
  - 文件路径
  - 位置或可定位片段
  - 问题类型：错别字 / 病句 / 重复 / 用词不稳 / 标点问题
  - 建议改法
- 用户确认前，不执行任何写入。
- 用户确认后，优先使用 `scripts/apply_replacements.py` 做精确替换，而不是手工大段改写。

## 标准流程

### 第一步：审校
读取目标文件，重点检查：
- 错别字、漏字、多字、近形字误用
- 搭配不当、主谓宾不顺、指代不清
- 句子过长导致的语义缠绕
- 明显重复、赘述、口语化失稳
- 标点误用和引号不统一

### 第二步：输出清单
输出格式固定为编号列表，例如：

1. `docs/content/chapters/zh/chapter-1.md`
   - 位置：`……原句片段……`
   - 问题：病句，主语缺失
   - 建议：改为 `……`

不要在这一轮直接贴整段重写，除非用户明确要求“直接润色整段”。

### 第三步：等待确认
只有在用户给出以下类型确认后才修改：
- “执行全部”
- “按 1、3、5 修改”
- “只改错别字，不动病句”
- “按你的建议执行”

### 第四步：脚本替换
确认后：
1. 生成一个 replacement JSON 文件到临时目录或仓库临时文件。
2. 使用 `scripts/apply_replacements.py` 执行替换。
3. 替换完成后回显成功修改的编号和文件。

## 替换规则
- 优先做“可精确定位的最小替换”，不要顺手重写整段。
- 同一个短语在同文件中出现多次时，必须指定 `count` 或拆成多条，避免误伤。
- 如果原文无法稳定定位，先回到人工确认，不要盲改。
- 如果建议涉及结构性重写，不适合脚本替换，应明确告诉用户该条需要手工改写。

## 脚本
- `scripts/apply_replacements.py`
  - 读取 replacement JSON
  - 校验每条替换是否命中
  - 按顺序执行替换
  - 对未命中、命中过多、重复编号等情况直接报错

## 适用文件
- `docs/content/**/*.md`
- `创作/**/*.md`
- `资料/**/*.md`
- 其他用户明确指定的 `.md`、`.txt`

## 输出要求
- 审校阶段：只给编号清单和建议，不改文件。
- 执行阶段：简要说明已执行哪些编号，并列出修改文件。
