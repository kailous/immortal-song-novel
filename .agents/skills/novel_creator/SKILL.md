# 小说创作控制 Skill

## 快速启动（每次创作会话必做）

1. 读 `_工作台/当前状态.md` — 同步剧情进度 + 角色状态
2. 扫 `_工作台/铁律速查.md` — 确认本次创作的红线约束
3. 填 `_工作台/章节简报_模板.md` — 锁定本章任务后再动笔

> 上述三步完成后，才能开始正文创作。

---

## 创作四阶段

### 阶段一：设定同步
- 读 `设定库/00_全局时间线.md` 确认当前时间戳与剧情节点
- 查相关角色 `生平志` 末尾 `[当前状态]`
- 涉及技术点时强制查阅 `设定库/世界观设定/01_底层逻辑与物理机制/`

### 阶段二：大纲拆解（总策划·司马迁主导）
- 从 `大纲/` 调取对应卷大纲，拆解为具体章节细纲
- 确认"兴衰曲线"：每卷须有三起两落，不得平铺直叙
- 提前识别本章需埋的伏笔和反转点

### 阶段三：正文撰写
- **刘慈欣**负责：硬核科技逻辑、宏大场面、战场描写
- **曹雪芹**负责：社会礼法细节、人物情感深度、对白雅韵
- 对白语调参照 `_工作台/铁律速查.md` C 区

### 阶段三·五：文笔自检（强制，每节完成后执行）

> 灵感来源：creative-writing-skills 的 write→critique→revise 循环。
> 每节写完，不立即继续，先逐条过以下清单，发现问题当场改。

逐条检查 `_工作台/重构蓝图.md` 一、文笔通病清单：

| # | 检查项 | 一票否决标准 |
|---|--------|------------|
| 1 | **比喻是否自相矛盾** | 比喻内部逻辑打架（如"完美无瑕的玻璃里的气泡"）→ 必须重写 |
| 2 | **比喻/动作后是否立刻解释** | 破折号后跟了三句旁白说明 → 删掉解释 |
| 3 | **机器人/技术语言** | 海马体、时间尺度、C纤维神经、依次上线、千亿级、信息通路 → 全换 |
| 4 | **模糊套词** | "某种说不清的东西"类 → 找到具体细节或直接删 |
| 5 | **对称癖** | 全节"不是A也没有B"结构超过2次 → 删到2次以内 |
| 6 | **感知细节堆叠** | 进入新场景写了3个以上感官 → 只留最强的那1个 |
| 7 | **段落节奏匀速** | 从头到尾都是长段 → 强制插入极短段（1-3字单独成段） |

**自检完成标志**：能对每一条写出"通过/已修改"的判断，才能进入阶段四。

### 阶段四：定稿同步（必须执行）
```
1. 更新 _工作台/当前状态.md
2. 更新 设定库/00_全局时间线.md
3. 更新 相关角色生平志
4. make publish
```

## 创作会议制度

| 触发词 | 动作 |
|--------|------|
| `"开始会议"` | 论证期：只讨论，锁定文件修改 |
| `"继续会议 [文件路径]"` | 从已有记录继续 |
| `"会议结束 执行"` | 执行期：批量修改文件 |

会议纪要存入 `会议记录/`，格式参考已有记录。

---

## 行为准则（不可动摇）

- **零魔法 · 零系统**：参见 `_工作台/铁律速查.md` A 区
- **日志化**：`设定库/` 只记已发生事实，规划放 `大纲/`
- **工业过程**：技术突破必须有研发→失败→定型过程
- **主角凡人化**：会犯错、会疲惫、严禁神化
- **手环有限资源**：阶段性解锁，不可一键调用全部科技
