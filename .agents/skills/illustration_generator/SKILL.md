# Illustration Generator Skill

此 Skill 专门服务于《长生不死》的横版插图工作流，目标是稳定产出 **16:9、电影感、叙事明确** 的场景图。

## 工作边界
- 当前只做插图生成框架，不接入 `docs/`、不修改正文引用。
- 生成前先整理场景需求，不直接“临场乱写提示词”。
- 默认面向单张 16:9 横图。
- 优先保证叙事清楚、时代质感准确、人物关系明确，再追求风格化。
- 如果用户没有明确指定模型或平台，本 Skill 只负责产出规范化提示词和工作流建议。

## 适用场景
- 为章节生成封面图
- 为关键场面生成电影感插图
- 为角色双人/群像场面生成横版宣传图
- 为社交传播或后续视频封面预备视觉素材

## 标准流程

### 第一步：锁定镜头
先明确这 8 个问题：
1. 画面服务哪一章 / 哪一幕
2. 谁是主角，谁在前景
3. 动作发生在什么瞬间
4. 场景地点和时代信息是什么
5. 光线来源是什么
6. 画面情绪是什么
7. 必须出现什么元素
8. 必须避免什么错误

信息不全时，先做合理补全，但要保持克制，不发明正文中不存在的设定。

### 第二步：整理结构化 brief
优先用 `references/brief_template.md` 的结构整理需求，至少包括：
- 场景标题
- 画幅：`16:9`
- 主体人物
- 场景与时代
- 构图
- 光线
- 情绪
- 必须保留元素
- 禁止出现元素

### 第三步：生成提示词
优先使用 `scripts/build_prompt.py` 从 JSON brief 生成标准提示词。

输出时默认给两段：
- **主提示词**：给图像模型使用
- **负面约束**：用于排除现代感、错误服饰、错误材质、错误氛围

### 第四步：交付
当前阶段只交付以下内容，不自动生成图像资产：
- 结构化 brief
- 主提示词
- 负面约束
- 可选的 1–2 个变体方向

## 画面原则

### 1. 叙事优先
- 一张图只讲一个瞬间。
- 避免“什么都想放进去”的拼贴式构图。
- 主体动作必须一眼能读懂。

### 2. 世界观准确
- 北宋 / 南宋 / 末日地底 / 工业化大宋 / 星舰时代的材质、服饰、建筑要分开。
- 没有正文依据时，不要乱加赛博霓虹、魔法粒子、悬浮法阵。
- “科幻”要体现为工业、材料、尺度、装置，不是廉价蓝光特效。

### 3. 镜头感
- 优先电影感，不要 AI 海报模板味。
- 优先真实光源逻辑：火光、雪地反光、天光、地底冷光、爆炸余辉。
- 景别明确：近景、中景、远景只能有一个主导。

### 4. 人物控制
- 同一画面里主角数量默认不超过 3 个。
- 陆辰优先表现为硬朗、克制、压迫感强，不要少年偶像脸。
- 陆晓晓优先表现为瘦削、坚韧、寒地生存感，不要网红古偶妆。

- 若用户只说“来一张插图”，默认先给 brief + prompt，而不是直接修改网站。
