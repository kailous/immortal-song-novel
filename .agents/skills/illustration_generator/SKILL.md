# Illustration Generator Skill

此 Skill 专门服务于《长生不死》的横版插图工作流，目标是稳定产出 **16:9、写实电影风格、叙事明确** 的场景图。

## 工作边界
- 当前只做插图生成与审核框架，不接入 `docs/`、不修改正文引用。
- 透明品牌模板与 logo 资产已经固定在 `资源/品牌/`，插图定稿后优先走脚本合成，不再手工摆放 logo。
- 生成前先整理场景需求，不直接“临场乱写提示词”。
- 默认面向单张 16:9 横图。
- 默认套用 `references/poster_template.md` 中定义的 **B 方案优化版** 海报模板。
- 当前 B 模板以 `资源/品牌/poster-template-overlay.png` 为固定透明叠层模板。
- 章节插图默认按章节目录落图：`docs/images/chapters/chapter-xxx/`。
- 跨章节复用或模板类素材放在 `docs/images/chapters/shared/`。
- 优先保证叙事清楚、时代质感准确、人物关系明确，再追求风格化。
- 人物外形必须尽量对齐图鉴与设定集里已有的公开形象图和角色描述。
- 固定风格为**写实电影风格**，不是插画风、动漫风、仙侠海报风。
- 如果用户没有明确指定模型或平台，本 Skill 至少要交付：`brief -> prompt -> prompt 审核 -> 生成后复核清单`。

## 适用场景
- 为章节生成封面图
- 为关键场面生成电影感插图
- 为角色双人/群像场面生成横版宣传图
- 为社交传播或后续视频封面预备视觉素材

## 标准流程

### 第一步：锁定镜头
先明确这 8 个问题：
1. 画面服务哪一章 / 哪一幕
2. 谁是主角，谁在前景
3. 动作发生在什么瞬间
4. 场景地点和时代信息是什么
5. 光线来源是什么
6. 画面情绪是什么
7. 必须出现什么元素
8. 必须避免什么错误

信息不全时，先做合理补全，但要保持克制，不发明正文中不存在的设定。

### 第二步：整理结构化 brief
优先用 `references/brief_template.md` 的结构整理需求，至少包括：
- 场景标题
- 画幅：`16:9`
- 主体人物
- 场景与时代
- 构图
- 光线
- 情绪
- 必须保留元素
- 禁止出现元素
- 角色参考来源

如果画面里出现已有角色，先查：
- `docs/content/profiles/zh/*.md`
- 角色条目里的公开形象图
- 如需稳定服装，先调用 `$costume_designer` 生成或读取该角色当前阶段的服装设定

不要凭空发明角色长相。

### 第三步：生成提示词
优先使用 `scripts/build_prompt.py` 从 JSON brief 生成标准提示词。

输出时默认给两段：
- **主提示词**：给图像模型使用
- **负面约束**：用于排除现代感、错误服饰、错误材质、错误氛围
- 若存在服装设定，应把 `outfit_id`、服装层次、必须保留元素和负面服装约束并入 prompt
- 若用户要求出海报，而不是纯插图，默认附加模板要求：按 B 版模板执行，右下角小尺寸 Logo 信息组、主视觉区占比 85% 以上、顶部和左侧保留呼吸感，不得擅自改成底部居中方案。

### 第四步：审核提示词
生成提示词后，不直接进入出图。先按 `references/review_checklist.md` 逐条审核：
- 是否符合章节情节
- 是否误加正文里没有的设定
- 是否保持 16:9 横版
- 是否明确写了写实电影风格
- 是否绑定了角色形象参考
- 是否绑定了稳定的服装设定
- 是否存在容易导致模型画错的模糊描述
- 是否明确了人物与地面、重心、视线、遮挡物之间的真实关系，避免人物像后贴上去

审核不通过时，先改 prompt，再进入生成。

### 第五步：进入生成
审核通过后，才进入生成阶段。

如果用户明确要求直接生成图片：
- 优先用当前审核通过的主提示词
- 如有人物，尽量同时引用对应角色图作为参考
- 仍然保持“单张图只讲一个瞬间”的原则
- 若未特别说明，默认使用 B 方案优化版海报模板输出
- 品牌成片优先使用 `scripts/apply_poster_template.py` 将 `资源/品牌/poster-template-overlay.png` 直接叠加到插图上，保持版式一致
- 若画面服务具体章节，默认把输出文件落到对应目录：
  - `docs/images/chapters/chapter-001/poster-main.webp`
  - `docs/images/chapters/chapter-001/scene-engine-room.webp`
  - `docs/images/chapters/chapter-001/scene-mountain-night.webp`
- 只有可跨章节复用的通用图，才放到 `docs/images/chapters/shared/`

### 第六步：生成后复核
生成后必须再检查：
- 人物是否和设定图像一致
- 年代、服饰、器物是否错位
- 动作是否符合情节
- 人物数量和关系是否画错
- 人物是否真实站在 / 蹲在 / 倒在场景里，而不是悬浮、拼接或重心失真
- 是否出现多余肢体、错误手部、错误饰品、错误武器
- 是否把写实电影风格画成海报风或游戏概念图
- 是否发生服装漂移或服装层次错误
- 是否符合默认海报模板的 logo 位置、大小、安全区和低干扰原则
- 是否仍然保留电影剧照壁纸感，而不是重新滑回传统网文封面

如果发现不一致或逻辑错误，先列出问题，再修正 prompt 进入下一轮。

## 画面原则

### 1. 叙事优先
- 一张图只讲一个瞬间。
- 避免“什么都想放进去”的拼贴式构图。
- 主体动作必须一眼能读懂。

### 2. 世界观准确
- 北宋 / 南宋 / 末日地底 / 工业化大宋 / 星舰时代的材质、服饰、建筑要分开。
- 没有正文依据时，不要乱加赛博霓虹、魔法粒子、悬浮法阵。
- “科幻”要体现为工业、材料、尺度、装置，不是廉价蓝光特效。

### 3. 镜头感
- 优先写实电影感，不要 AI 海报模板味。
- 优先真实光源逻辑：火光、雪地反光、天光、地底冷光、爆炸余辉。
- 景别明确：近景、中景、远景只能有一个主导。

### 3.5 海报模板
- 默认模板为基于官方 VI 的 B 方案优化版。
- logo 是识别层，不是主视觉层。
- logo 版本优先取自官方单色 / 反白 / 极简字标，不临时自造变体。
- 默认按 B 版模板使用右下角小 Logo 信息组。
- 章节标题和短引语只作为辅助信息，不得压过画面主体。
- 主画面始终优先于版式装饰。

### 4. 人物控制
- 同一画面里主角数量默认不超过 3 个。
- 陆辰优先表现为硬朗、克制、压迫感强，不要少年偶像脸。
- 陆晓晓优先表现为瘦削、坚韧、寒地生存感，不要网红古偶妆。
- 如果角色已有公开形象图，优先对齐那张图的五官方向、年龄感、体态和衣着层级。
- 若图像结果与角色参考明显不符，应判定为失败稿，不直接采用。
- 若角色已有阶段服装设定，优先对齐对应 outfit_id，不随意换装。
- 人物必须与镜头、地形、光源、阴影、前景遮挡形成统一透视关系，严禁“像被拼进背景”的构图失败稿。

## 资源
- `references/brief_template.md`：结构化场景需求模板
- `references/poster_template.md`：默认 16:9 海报模板规范
- `references/style_guide.md`：本项目插图风格约束
- `references/review_checklist.md`：提示词审核与出图后复核清单
- `scripts/build_prompt.py`：从 JSON brief 生成标准提示词
- `scripts/apply_poster_template.py`：把透明插图模板直接叠加到章节插图上
- `资源/品牌/poster-template-overlay.png`：当前统一叠加用透明模板
- `资源/品牌/LOGO/`：当前正式 logo 导出资产
- 需要稳定服装时，先配合 `$costume_designer`

## 注意事项
- 暂时不要把图片插入文章或 `docs/`。
- 暂时不要默认生成竖图、头像、立绘，除非用户明确指定。
- 若用户只说“来一张插图”，默认先给 brief + prompt，而不是直接修改网站。
- 若用户要求出人物图，先读对应角色档案和角色图，再写 prompt。
- 若用户没有指定保存路径，但画面明确属于某一章，默认按 `chapter-xxx` 目录落图并沿用统一命名。
