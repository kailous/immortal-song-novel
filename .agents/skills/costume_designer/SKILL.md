# Costume Designer Skill

此 Skill 专门负责《长生不死》角色服装设定，目标是让角色在不同插图、章节场景和宣传图中保持稳定、可追踪的服装逻辑。

## 工作边界
- 先做服装设定，再做插图，不反过来。
- 当前阶段只生成服装设定与服装提示词片段，不直接改正文或 `docs/` 页面。
- 服装必须服务角色身份、时代、气候、行动功能，不做脱离剧情的“好看优先”设计。
- 每个角色服装设定都应支持后续被 `illustration_generator` 直接引用。

## 适用场景
- 为主角建立稳定服装方案
- 为角色在不同剧情阶段生成换装方案
- 为双人/群像插图统一服装层级
- 为后续插图生成提供稳定的服装 reference

## 标准流程

### 第一步：锁定角色与阶段
先明确：
1. 角色是谁
2. 所处年代 / 阵营 / 阶段
3. 服装用于什么场景
4. 当时气候与环境是什么
5. 角色是否需要机动、战斗、潜伏、劳动、仪式展示

### 第二步：读取角色依据
优先读取：
- `docs/content/profiles/zh/*.md`
- 角色公开形象图
- 当前章节或场景 brief

服装不能脱离角色已有外形和身份设定。

### 第三步：生成结构化服装设定
优先使用 `references/wardrobe_template.md` 整理，至少包括：
- outfit_id
- 角色名
- 阶段 / 用途
- 时代
- 内层 / 中层 / 外层
- 鞋履 / 腰带 / 配件
- 材质
- 配色
- 磨损度
- 必须保留元素
- 禁止出现元素

### 第四步：生成服装提示词片段
优先使用 `scripts/build_costume_prompt.py` 从 JSON wardrobe 生成：
- **服装描述片段**
- **负面服装约束**

它们不是完整插图 prompt，而是给 `illustration_generator` 拼接用的稳定组件。

### 第五步：一致性审核
生成后按 `references/consistency_checklist.md` 审核：
- 是否符合角色身份
- 是否符合时代
- 是否符合行动功能
- 是否和公开形象图冲突
- 是否会导致后续模型频繁画错

## 设计原则

### 1. 身份优先
- 服装先讲身份，再讲美感。
- 商贾遗孤、山村寡妇、斥候、将领、工匠、皇族，衣着逻辑必须分明。

### 2. 功能优先
- 需要奔跑、负重、潜伏、作战的角色，衣服不能设计成礼仪摆拍装。
- 天寒地冻、泥地、山路、军营、工坊的服装应有真实磨损。

### 3. 时代优先
- 宋代服装不能混入明清、影视古偶、仙侠元素。
- 科幻阶段也必须服从本书的工业感和材料逻辑。

### 4. 一致性优先
- 同一角色在同一阶段默认有一套主服装设定。
- 如果是换装，必须明确是“何时换、为什么换、哪些元素不变”。

## 与插图 Skill 的协作
- `illustration_generator` 在出有人物的图前，应优先调用本 Skill 的服装设定结果。
- 插图 prompt 应明确引用：
  - outfit_id
  - 服装核心层次
  - 必须保留元素
  - 负面服装约束
- 若插图结果服装走样，应先回查本 Skill 的设定是否不够明确，再收紧 prompt。

## 资源

- `references/style_rules.md`：本项目服装设计约束

## 注意事项
- 不要把角色服装做成古偶视觉设定。
- 不要默认“越华丽越好”，除非角色身份和场景明确要求。
- 若用户只说“给某角色做服装”，默认先产出主服装设定，不直接生成插图。
